# GOTTH Migration Guide: From Next.js to Go

This document outlines the step-by-step process for migrating the **Wandering Mind** blog from its current Next.js/React foundation to a high-performance **GOTTH** (Go, Templ, Tailwind, HTMX) stack.

## 1. Visual Foundation (The Newspaper Aesthetic)
The "soul" of this project lies in the `globals.css` file. 

- **Action**: Copy the contents of `src/app/globals.css` into `blog-gotth/static/css/styles.css`.
- **Note**: Since you are using Tailwind v4, you can use the `@theme` block to define your newspaper color palette and typography tokens.
- **Fonts**: Ensure the Google Font imports from the original CSS are preserved in the new `styles.css`.

## 2. Component Translation (React → Templ)
You need to recreate your React UI components as type-safe `.templ` files in `blog-gotth/templates/`.

| Next.js Component | GOTTH Equivalent |
| :--- | :--- |
| `src/components/Header.js` | `templates/header.templ` |
| `src/components/FeaturedArticle.js` | `templates/featured.templ` |
| `src/components/PostCard.js` | `templates/postcard.templ` |
| `src/app/layout.js` | `templates/layout.templ` (Already started) |

> [!TIP]
> Use Go structs to pass data into your Templ components for full type safety.

## 3. Data Layer (Markdown Parsing)
Next.js uses `gray-matter`. In Go, you will use:
- **Parser**: [goldmark](https://github.com/yuin/goldmark) for converting Markdown to HTML.
- **Frontmatter**: [frontmatter](https://github.com/adrg/frontmatter) or similar for extracting metadata.

### Migration Logic:
1. Create a `internal/posts` package.
2. Implement a function to read files from `../src/content/posts` (you can point the Go app to the same content directory).
3. Sort and filter posts exactly like `src/lib/posts.js` does.

## 4. Interactivity (htmx)
Identify parts of the site that currently use React `useState` or `useEffect` and replace them with htmx attributes.

- **Load More**: Use `hx-get` and `hx-swap="beforeend"` to append new post cards.
- **Search/Filters**: Use `hx-post` and `hx-trigger="keyup changed delay:500ms"` for instant search.
- **Auto-Refresh**: Use `hx-trigger="every 30s"` for news tickers.

## 5. SEO & Metadata
Next.js handles metadata via `generateMetadata`. In GOTTH, you’ll handle this manually in your `Layout` component.

- **Action**: Pass an `SEO` struct into your `templates.Layout` call.
- **Implementation**:
  ```go
  type SEO struct {
      Title       string
      Description string
      OGImage     string
  }
  ```

## 6. Vercel Deployment
The `blog-gotth/api/index.go` and `vercel.json` are already set up to handle routing. 

**Build Command on Vercel:**
You will need to tell Vercel how to build your project. In the Vercel Dashboard, set the build command to:
```bash
./bin/templ generate && npx @tailwindcss/cli -i static/css/styles.css -o static/css/output.css
```

## 7. Migration Checklist

- [ ] Map the `/post/[slug]` route in `main.go`.
- [ ] Verify htmx fragments are returning partial HTML (no layouts).
- [ ] Test the Vercel deployment via `vercel dev`.
